📖 Quick Start

Make a simple POST request to download content from any supported platform:

curl -X POST https://detachbackend-production.up.railway.app/api/download \ -H "Content-Type: application/json" \ -d '{ "url": "https://www.instagram.com/reel/DMw7_HDusQY/", "format": "video", "quality": "best" }'

📋 Request Parameters

url (required): The social media URL you want to download
format (optional): "video" or "audio" - defaults to "video"
quality (optional): "best" or "worst" - defaults to "best"

✅ Success Response Example

{ "success": true, "filePath": "/app/downloads/Video_by_snoopdogg_1754708049649.mp4", "filename": "Video_by_snoopdogg_1754708049649.mp4", "fileSize": 8418029, "platform": "instagram", "title": "Video by snoopdogg", "duration": 76, "uploader": "snoopdogg", "contentType": "video", "detectedPlatform": "instagram", "routedTo": "/api/instagram" }

📊 Platform-Specific Details

✅ Fully Working Platforms (8/10)
  • TikTok: Downloads videos and images (some may be blocked by platform)
  • LinkedIn: Full video downloads with metadata
  • Instagram: Reels, posts, and stories
  • Facebook: Public videos and posts
  • Pinterest: Videos and images from pins
  • Apple Podcasts: Full episode downloads
  • Spotify: Track downloads (where available)
  • X (Twitter): Videos and posts
⚠️ Limited Platforms (2/10)
  • YouTube: Enhanced bot detection causing failures
  • YouTube Music: Same detection issues as YouTube

We're actively working on resolving YouTube-related issues.

📈 Rate Limits

Rate Limit: 100 requests per hour per IP address
Timeout: 2 minutes per request
File Retention: Downloaded files are automatically cleaned up after 1 hour
